What is pentest-ai?

pentest-ai is the leading open-source AI pentest tool. It turns Claude Code into a full offensive security research environment with 28 specialized AI agents. 6 of these are Tier 2 agents that can execute recon, vulnerability scanning, web app testing, Active Directory attacks, exploit chain validation, and PoC verification directly, with your approval on every command. New in v3.0: autonomous exploit chaining that turns low-severity findings into full compromise, zero false positives via automated PoC validation, agentic swarm orchestration that coordinates parallel agent teams like a real red team, business logic flaw detection that catches what scanners miss, and CI/CD pipeline integration for continuous automated red teaming. Ask Claude anything security-related and it automatically routes to the right specialist. Every response is mapped to MITRE ATT&CK and paired with defensive guidance.

Autonomous Exploit Chaining

Chains low-severity findings into full system compromise. The exploit-chainer agent finds the info leak, combines it with weak permissions, and walks you through gaining admin access step by step. Each pivot requires your approval.

PoC-Validated Findings

The poc-validator agent automatically generates safe PoC scripts and executes them to verify every finding. Unconfirmed bugs never reach your report. Batch-validates entire scan reports in priority order.

Agentic Swarm Orchestration

The swarm-orchestrator coordinates all 28 agents as a parallel red team. Delegates recon, scanning, exploitation, and reporting to specialists. Tracks progress across a 7-phase engagement lifecycle.

Business Logic Flaw Detection

The bizlogic-hunter finds price manipulation, workflow bypasses, race conditions, and authorization logic flaws that standard scanners completely miss. Thinks like a fraudster to break your business rules.

CI/CD Pipeline Integration

The cicd-redteam agent generates ready-to-use pipeline configs for GitHub Actions, GitLab CI, and Jenkins. 3-tier scanning with configurable security gates that block deploys on critical findings.

MITRE ATT&CK Mapped

Every technique is cross-referenced with ATT&CK IDs. Know exactly where each finding sits in the adversary framework.

Dual Perspective

What's New in v3.0

5 new agents that transform pentest-ai from an advisory tool into an autonomous red team platform.

exploit-chainer: Autonomous Multi-Step Exploit Chains

Most scanners report isolated findings. The exploit-chainer takes those findings and automatically connects them into attack paths that demonstrate real compromise. It chains an information disclosure with a weak permission, adds a credential reuse, and walks you from a low-severity info leak to Domain Admin. Every pivot point requires your approval before executing. Five chain types: Information Leak to Full Compromise, Chained Web Vulnerabilities, AD Privilege Escalation, Cloud Pivot Chains, and Cross-Environment Chains. Each step includes MITRE ATT&CK mapping, OPSEC tags, and failure fallbacks.

poc-validator: Zero False Positives via Verified PoCs

Scanners produce noise. The poc-validator eliminates it by generating safe, non-destructive Proof of Concept scripts for every finding and executing them against the target. It uses canary values instead of real payloads, never creates persistence mechanisms, and includes timeouts on every script. Batch validation mode processes entire scan reports: Critical findings first, then High, then deduplication across hosts. Common false positive patterns are automatically detected: version-only detection, WAF interference, dead code paths, mitigating controls, configuration-dependent, and OS/platform mismatch.

swarm-orchestrator: Your AI Red Team Manager

Run a full penetration testing engagement with a single instruction. The swarm-orchestrator manages all 28 agents through a 7-phase lifecycle: Planning, Reconnaissance (parallel workstreams), Vulnerability Assessment, Exploitation, Post-Exploitation, Detection Engineering, and Reporting. It delegates tasks to the right specialist, runs independent workstreams in parallel, handles agent-to-agent handoffs, resolves conflicts between agent outputs, and compiles everything into a unified report. Real-time dashboard shows agent status, findings summary, attack chain progress, and objective tracking.

bizlogic-hunter: Catching What Scanners Miss

Standard vulnerability scanners test for known technical flaws. The bizlogic-hunter tests for logical errors: the shopping cart that lets users set their own price, the workflow that can be completed out of order, the race condition that allows double-spending. It covers 8 categories: Price/Payment Manipulation, Authentication Logic, Authorization/Access Control, Workflow/State Bypass, Race Conditions, Data Validation Logic, Feature Abuse/Rate Limit Bypass, and API-Specific Logic Flaws. Maps the intended application workflow first, then systematically breaks it.

cicd-redteam: Security on Every Code Push

Point-in-time pentests are outdated. The cicd-redteam agent integrates directly into your deployment pipeline. Tier 1 scans run on every push in under 5 minutes (secrets, dependencies, SAST, IaC). Tier 2 scans run on every PR (container, API, DAST). Tier 3 scans run on schedule (full DAST, network, cloud audit). Configurable security gates block merges when critical issues are found. Trend tracking across runs shows whether your security posture is improving or regressing. Ready-to-use configs for GitHub Actions, GitLab CI, and Jenkins.

Upgraded Agent Ecosystem

v3.0 brings the total to 28 agents and 6 Tier 2 execution agents. The new agents integrate seamlessly with the existing ecosystem: vuln-scanner feeds findings to poc-validator for verification, verified findings flow to exploit-chainer for chain building, the swarm-orchestrator coordinates everything, and bizlogic-hunter catches the logic flaws that no other agent or scanner would find. The cicd-redteam agent brings the entire pipeline into your CI/CD workflow for continuous security.

Frequently Asked Questions

Common questions about pentest-ai.

Do I need security certifications to use pentest-ai?
No certifications are required to install and explore the agents. However, for real engagements, you should have proper training and authorization. The agents adapt to your skill level. Beginners get explanations, experienced operators get exact command syntax.
Does pentest-ai execute attacks or access systems?
6 Tier 2 agents (recon-advisor, vuln-scanner, web-hunter, ad-attacker, exploit-chainer, poc-validator) can execute reconnaissance, enumeration, and validation commands directly, but every command goes through Claude Code's permission prompt first. You see the full command and approve or deny it. The remaining 22 agents provide methodology guidance, analysis, and documentation only. No agent generates functional exploit code or bypasses the permission system.
Which Claude subscription do I need?
You need Claude Pro or Max with Claude Code enabled. The agents use the Sonnet model by default, which is included with both subscription tiers.
Can I customize the agents or create new ones?
Yes. Agents are plain Markdown files with YAML frontmatter. You can modify existing agents, change their model, add tools, or create entirely new agents. See the Customization guide for details.
How does automatic agent routing work?
Claude Code reads the description field in each agent's YAML frontmatter. When you describe a task, Claude matches your intent to the most relevant agent and delegates automatically. You can also invoke agents by name for direct control.
Can I use pentest-ai for bug bounty programs?
Yes, within the program's scope and rules. The dedicated bug-bounty agent handles methodology, target selection, and report writing for HackerOne, Bugcrowd, and Intigriti. The bizlogic-hunter agent finds business logic flaws that other hunters miss. Use the web-hunter for active testing, recon-advisor for enumeration, and report-generator for professional write-ups. Always operate within the program's rules of engagement.
How is pentest-ai different from other AI pentest tools?
pentest-ai is methodology-first with execution when you want it. Most AI pentesting frameworks wrap tools behind an AI execution layer. pentest-ai starts with methodology: what to do, why it works, how defenders catch it. Key differentiators: autonomous exploit chaining (chains low-severity findings into full compromise), zero false positives (PoC-validated findings only), agentic swarm orchestration (parallel agent teams), business logic flaw detection (catches what scanners miss), and CI/CD integration (continuous red teaming). Zero infrastructure required: just copy markdown files.
What is autonomous exploit chaining?
The exploit-chainer agent takes isolated, often low-severity findings from scans and connects them into multi-step attack paths that demonstrate full system compromise. For example, it might chain an information disclosure bug with a weak permission setting to walk you through gaining admin access. Each step requires your approval before proceeding. It adapts when paths are blocked, finding alternative routes to the objective.
Can pentest-ai integrate into CI/CD pipelines?
Yes. The cicd-redteam agent generates ready-to-use pipeline configurations for GitHub Actions, GitLab CI, and Jenkins. It provides 3-tier scanning (per-push for secrets and SAST, per-PR for container and DAST scans, scheduled for comprehensive assessments) with configurable security gates that block deployments when critical issues are found.

Installation

One command. No dependencies. No build tools.

New to Claude? No problem. The setup guide walks you through creating an account, installing the CLI, and running your first agent in about 5 minutes.

One-liner install (recommended)

Or use the interactive installer
./install.sh # Choose: global, project-level, uninstall, or check status
Then open Claude Code and try
# Just describe your task naturally "Scan 10.10.1.0/24 and build me an attack chain to Domain Admin"
