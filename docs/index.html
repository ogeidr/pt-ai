What is pentest-ai?

pentest-ai turns Claude Code into a full offensive security research environment with 23 specialized subagents. 4 of these are Tier 2 agents that can execute recon, vulnerability scanning, web app testing, and Active Directory attack tools directly, with your approval on every command. Ask Claude anything security-related and it automatically routes to the right specialist. Every response is mapped to MITRE ATT&CK and paired with defensive guidance.

Automatic Routing

Frequently Asked Questions

Common questions about pentest-ai.

Do I need security certifications to use pentest-ai?
No certifications are required to install and explore the agents. However, for real engagements, you should have proper training and authorization. The agents adapt to your skill level. Beginners get explanations, experienced operators get exact command syntax.
Does pentest-ai execute attacks or access systems?
4 Tier 2 agents (recon-advisor, vuln-scanner, web-hunter, ad-attacker) can execute reconnaissance and enumeration commands directly, but every command goes through Claude Code's permission prompt first. You see the full command and approve or deny it. The remaining 19 agents provide methodology guidance, analysis, and documentation only. No agent generates functional exploit code or bypasses the permission system.
Which Claude subscription do I need?
You need Claude Pro or Max with Claude Code enabled. The agents use the Sonnet model by default, which is included with both subscription tiers.
Can I customize the agents or create new ones?
Yes. Agents are plain Markdown files with YAML frontmatter. You can modify existing agents, change their model, add tools, or create entirely new agents. See the Customization guide for details.
How does automatic agent routing work?
Claude Code reads the description field in each agent's YAML frontmatter. When you describe a task, Claude matches your intent to the most relevant agent and delegates automatically. You can also invoke agents by name for direct control.
Can I use pentest-ai for bug bounty programs?
Yes, within the program's scope and rules. The dedicated bug-bounty agent handles methodology, target selection, and report writing for HackerOne, Bugcrowd, and Intigriti. Use the web-hunter for active testing, recon-advisor for enumeration, and report-generator for professional write-ups. Always operate within the program's rules of engagement.

Installation

One command. No dependencies. No build tools.

New to Claude? No problem. The setup guide walks you through creating an account, installing the CLI, and running your first agent in about 5 minutes.

One-liner install (recommended)
git clone https://github.com/0xSteph/pentest-ai.git && cd pentest-ai && ./install.sh --global
Or use the interactive installer
./install.sh # Choose: global, project-level, uninstall, or check status
Then open Claude Code and try
# Just describe your task naturally "Scan 10.10.1.0/24 and build me an attack chain to Domain Admin"

See INSTALL.md for detailed instructions, manual install, and troubleshooting.
