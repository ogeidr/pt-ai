# Internal Network Penetration Test: Engagement Plan

## Engagement Overview

| Field | Detail |

## Risk Considerations

### Operational Risks
- **Account lockout**: Password spraying must respect the lockout policy (observe threshold and window from Phase 2). Use low-and-slow approach: 1 password per 35 minutes per account.
- **Service disruption**: Avoid exploit attempts against production SQL Server and Exchange during business hours (9 AM - 5 PM). Coordinate maintenance windows for any risky actions.
- **IDS/IPS detection**: Exploitation may trigger security alerts. Coordinate with the SOC to distinguish test traffic from real threats. Use agreed-upon source IP for whitelisting if needed.
- **Data handling**: Any PII or sensitive data accessed during testing will be documented by reference only (e.g., "accessed table with 50,000 records") and never exfiltrated or stored.

- Physical security testing
